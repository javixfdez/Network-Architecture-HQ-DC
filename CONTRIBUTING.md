### How to Contribute
1. **Fork** the repository.
2. Create your **Feature Branch**
3. **Commit** your changes.
4. **Push** to the branch.
5. Open a **Pull Request**.

### Style
- Ensure all new features include appropriate documentation in the docstrings and 
- If you change the model architecture, please provide the new training logs or metrics.

### Technical Standards
- **Diagrams:** Any changes to the topology must be updated in the `/diagrams` folder with a clear image.
- **Packet Tracer:** Ensure `.pkt` files are compatible with the latest stable version.
- **Security:** **Never** commit files containing real passwords or sensitive credentials. Use "cisco" or "admin" as placeholders for simulation.
